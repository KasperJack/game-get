## related projects / research

looked at a few existing tools while designing this.

### Project GLD
https://github.com/Y0URD34TH/Project-GLD

GUI game downloader that uses:
- IGDB for metadata
- Lua scripts as pluggable backends per source

also uses the Hydra source format (JSON-based).

good reference for backend architecture.

### Hydra
https://github.com/hydralauncher/hydra

game launcher with pluggable download sources.

worth studying more for how sources are defined and managed.

### cs.rin.ru
https://cs.rin.ru/forum

massive forum with information and downloads for most cracked games.

not directly usable as an automated source, but useful as a reference database.

### CrackWatch
https://crackwatch.com/api/games?slug={slug}

used to provide a crack status API, but it's now deprecated.
